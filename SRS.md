

# **Спецификация требований к программному обеспечению (SRS)**

## **Коллаборативный графический редактор**

## **1. Введение**

### **1.1. Назначение и область применения**

Настоящий документ определяет требования к программному обеспечению, представляющему собой веб-ориентированный коллаборативный графический редактор, предназначенный для совместного создания и редактирования графических материалов несколькими пользователями в режиме реального времени.

Программный продукт ориентирован на использование в образовательных, проектных и исследовательских целях, а также для проведения удалённых мозговых штурмов и коллективной визуализации идей.

**Функциональные возможности системы включают:**

* предоставление веб-интерфейса для создания и редактирования графических холстов;
* поддержку базовых инструментов графического редактирования (рисование, геометрические фигуры, текст, выбор и перемещение объектов);
* обеспечение синхронизации изменений между участниками сессии в режиме реального времени;
* управление пользователями и их правами доступа;
* сохранение истории действий в рамках активной сессии;
* экспорт результата работы в стандартные графические форматы.

**Функциональность, не входящая в рамки системы:**

* реализация расширенных возможностей профессиональных графических редакторов (многоуровневые слои, сложные фильтры, цветокоррекция);
* работа с трёхмерной графикой или видеоконтентом;
* долговременное версионное хранение проектов;
* поддержка голосовой и видеосвязи;
* функционирование без подключения к сети Интернет.

**Цели разработки:**

* обеспечение удобного инструмента для удалённого совместного графического взаимодействия;
* повышение эффективности командной работы за счёт мгновенной синхронизации изменений;
* снижение порога входа для пользователей за счёт простого и интуитивного интерфейса.

### **1.2. Термины, определения и сокращения**

| Термин             | Описание                                                                                           |
| ------------------ | -------------------------------------------------------------------------------------------------- |
| **Холст (Canvas)** | Графический документ, являющийся основной рабочей областью                                         |

| **WebSocket**      | Протокол двусторонней связи для передачи данных в реальном времени                                 |

## **2. Общее описание системы**

### **2.1. Контекст и взаимодействие компонентов**

Программное обеспечение реализуется в виде одностраничного веб-приложения и включает следующие логические компоненты:

* клиентскую часть, выполняемую в браузере пользователя;
* сервер приложений, реализующий бизнес-логику и управление пользователями;
* сервер реального времени, обеспечивающий синхронизацию операций между клиентами;
* систему хранения данных для учётных записей и метаданных холстов.

---

### **2.2. Основные функции системы**

* регистрация и аутентификация пользователей;
* создание, открытие и управление графическими холстами;
* выполнение операций графического редактирования;
* совместная работа в режиме реального времени;
* отображение присутствия участников;
* экспорт текущего состояния холста в файл.

---

### **2.3. Классы пользователей**

| Категория пользователя           | Описание                                                                  |
| -------------------------------- | ------------------------------------------------------------------------- |
| Гость                            | Может пройти регистрацию или аутентификацию                               |
| Аутентифицированный пользователь | Может создавать холсты и участвовать в совместной работе                  |
| Редактор                         | Имеет право изменять содержимое холста                                    |
| Наблюдатель                      | Может просматривать изменения и использовать чат без права редактирования |

---

### **2.4. Условия эксплуатации**

* клиентская часть функционирует в современных браузерах с поддержкой HTML5, Canvas API и WebSocket;
* серверная часть развёртывается в среде с поддержкой контейнеризации;
* для работы требуется стабильное подключение к сети Интернет.

---

### **2.5. Ограничения и допущения**

* система ориентирована на работу в браузере и не предусматривает нативные клиенты;
* корректная работа зависит от пропускной способности сети;
* используется стороннее программное обеспечение с открытым исходным кодом.

## **3. Требования к системе**

### **3.1. Функциональные требования**

| ID   | Требование                                                   |
| ---- | ------------------------------------------------------------ |
| FR-1 | Система должна обеспечивать регистрацию и вход пользователей |
| FR-2 | Пользователь должен иметь возможность создать новый холст    |
| FR-3 | Владелец холста должен управлять доступом участников         |
| FR-4 | Редактор должен иметь доступ к инструментам рисования        |
| FR-5 | Все изменения должны синхронизироваться в реальном времени   |
| FR-6 | Система должна отображать активных участников                |
| FR-7 | Холст должен экспортироваться в PNG, JPEG или SVG            |

---

### **3.2. Нефункциональные требования**

#### **Производительность**

* время отклика интерфейса — не более 100 мс;
* задержка синхронизации — не более 500 мс;
* поддержка до 50 одновременных пользователей на одном холсте.

#### **Надёжность**

* доступность сервиса — не менее 99,5%;
* автоматическое переподключение при потере связи.

#### **Безопасность**

* хранение паролей в хэшированном виде;
* разграничение прав доступа;
* использование защищённых каналов передачи данных.

#### **Удобство использования**

* минимальное время освоения;
* соответствие базовым требованиям доступности.

#### **Сопровождаемость**

* модульное тестирование;
* централизованное логирование ошибок.

---

### **3.3. Интерфейсные требования**

* веб-интерфейс с минималистичным дизайном;
* REST API для управления данными;
* WebSocket-интерфейс для синхронизации.

---

### **3.4. Требования к данным**

**Основные сущности:**

* User

**Требования:**

* обеспечение ссылочной целостности;

* регулярное резервное копирование данных.
